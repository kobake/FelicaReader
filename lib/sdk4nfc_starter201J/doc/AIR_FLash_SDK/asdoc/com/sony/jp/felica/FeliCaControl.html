FeliCa アクセスコントロールクラス.

このクラスはNFC Proxy Service との通信、イベント送受信などを行います。

AIR/Flash アプリケーションからの要求(Request)に対する NFC Proxy Service の 返答(Response)は非同期に行われ、Response の受信は Event により通知されます。

全ての NFC Proxy Service との通信に先んじて、open() によってAIR/Flash アプリケーションと NFC Proxy Service 間の接続を確立する必要があります。 open() の成否は OpenStatusEvent によって通知されます。そのため、open() 呼び出し前に OpenStatusEvent の各イベントタイプに対して addEventListener() によってイベントリスナー を登録しておく必要があります。

全ての NFC Proxy Service との通信は、各Requestクラス(例：feliCaOpenReaderWriterAutoRequest) のオブジェクトを access() の引数に指定して実行することで開始されます。Request の結果の Response は FeliCaStatusEvent によって通知されます。そのため、access() 呼び出しの前に、 FeliCaStatusEvent の各イベントタイプに対して addEventListener() によってイベントリスナー を登録しておく必要があります。

FeliCa への一連のアクセスが完了した後、NFC Proxy Service との接続を終了させるために close() を呼び出てください。

access()method
public function access(request:Object):void

FeliCa リーダー／ライターまたはカードへのアクセスを行います.

引数に渡されたRequest(例：FeliCaPollingAndGetCardInformationRequest クラスのオブジェクト) をその設定されたプロパティに従って実行します。

access() 呼び出しの後、NFC Proxy Service との接続の成否は、FeliCaStatusEvent によって 非同期に通知されます。そのため、access() 呼び出しの前にこれらのイベントタイプに対して addEventListener() によってイベントリスナーを登録してください。

access() の呼び出しに先んじんて、open() の呼び出しによってNFC Proxy Service との接続が確立 されている必要があります。

Parameters
request:Object — FeliCa アクセスする為の情報を保持するオブジェクトです。(XXXRequest クラスのオブジェクトを指定してください。)

addEventListener()method 
public function addEventListener(type:String, listener:Function, useCapture:Boolean = false, priority:int = 0, useWeakReference:Boolean = false):void

イベントリスナーオブジェクトを EventDispatcher オブジェクトに登録し、リスナーがイベントの通知を受け取るようにします。

Parameters
type:String — イベントのタイプです。FeliCaStatusEvent クラスもしくは OpenStatusEvent クラスのイベントタイプを指定してください。
 
listener:Function — イベントを処理するリスナー関数です。
 
useCapture:Boolean (default = false) — リスナーが、キャプチャ段階、またはターゲットおよびバブリング段階で動作するかどうかを判断します。useCapture を true に設定すると、リスナーはキャプチャ段階のみでイベントを処理し、ターゲット段階またはバブリング段階では処理しません。 useCapture を false に設定すると、リスナーはターゲット段階またはバブリング段階のみでイベントを処理します。 3 つの段階すべてでイベントを受け取るには、addEventListener() を 2 回呼び出します。useCapture を true に設定して 1 度呼び出し、useCapture を false に設定してもう一度呼び出します。
 
priority:int (default = 0) — イベントリスナーの優先度レベルです。優先度は、32 ビット整数で指定します。数値が大きくなるほど優先度が高くなります。優先度が n のすべてのリスナーは、優先度が n-1 のリスナーよりも前に処理されます。複数のリスナーに対して同じ優先度が設定されている場合は、追加された順番でリスナーが処理されます。デフォルトの優先度は 0 です。
 
useWeakReference:Boolean (default = false) — リスナーへの参照が強参照と弱参照のいずれであるかを判断します。デフォルトである強参照の場合は、リスナーのガベージコレクションが回避されます。弱参照では回避されません。

Throws
— 指定された listener は関数ではありません。
